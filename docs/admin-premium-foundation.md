# Premium Admin Foundation (Этап 1)

## Цель этапа

Заложить фундамент premium mobile-first админ-панели для собственника бизнеса, чтобы уже на первом экране было понятно:
- сколько денег сегодня;
- где риски потерь;
- какие действия важны прямо сейчас.

Этап не закрывает весь Variant C. Он задает структуру, UX-правила, дизайн-примитивы и каркасы экранов, на которые дальше добавляются метрики и бизнес-сценарии.

## Информационная архитектура

### Первый уровень (нижняя навигация)

1. `Главная`
2. `Заказы`
3. `Клиенты`
4. `Рассылки`
5. `Ещё`

### Разделы внутри `Ещё`

- `Поддержка`
- `Аналитика` (foundation-заготовка)
- `Темы клиентов` (foundation-заготовка)
- `Система` / наблюдаемость (вторично)
- `Флаги` (внутри `Система`)
- `Аудит действий` (внутри `Система`)

Принцип: на первом уровне только то, что влияет на деньги и удержание клиентов. Служебные контуры скрыты глубже.

## Почему структура оптимальна для владельца

- Владелец получает ответы за 3 секунды: деньги, заказы, потери, что делать.
- Основные сценарии в 1-2 тапа из нижней навигации.
- Нет перегруза техническими разделами на верхнем уровне.
- Системные разделы доступны, но не отвлекают от операционного фокуса.

## Mobile-first принципы

- Базовый viewport: ширина `390-430px`.
- Основная навигация: sticky `bottom nav`.
- Учет `safe-area` (`env(safe-area-inset-*)`) для iOS/Android.
- Корректная высота в webview через CSS-переменную `--app-height`, синхронизированную с `window.innerHeight`.
- Крупные tap targets: минимум `44px`.
- Ключевые экраны без горизонтального скролла.
- Карточная подача данных вместо desktop-таблиц.

## Navigation shell foundation

Реализован mobile shell:
- верхняя sticky-шапка с заголовком и контекстом экрана;
- контентная зона карточек и секций;
- фиксированная нижняя навигация из 5 бизнес-разделов;
- sticky action bar на ключевых сценариях;
- Telegram WebApp инициализация (`ready`, `expand`) и mobile viewport-адаптация.

## Визуальная логика «продающей» панели

На уровне foundation заложены паттерны:
- `Сегодня`
- `Сравнение с прошлым периодом`
- `Требует внимания`
- `Потери`
- `Точки роста`
- `Быстрые действия`
- `Инсайт дня`

Пользовательский язык — бизнес-ориентированный, без техжаргона на основных экранах.

## Data/Backend foundation

Без тяжелого backend rework добавлен lightweight endpoint:
- `GET /api/admin/mobile-summary`

Он агрегирует данные для главной:
- hero KPI за сегодня;
- сравнение с прошлым периодом;
- блоки `Требует внимания`, `Потери`, `Точки роста`;
- `Инсайт дня`;
- компактный 7-дневный тренд.

## Осознанные компромиссы текущего шага

- Часть аналитики пока агрегирована и упрощена, без глубоких разрезов.
- Экран `Аналитика` и `Темы клиентов` реализованы как foundation-каркасы, не как полный функционал Variant C.
- В системных разделах сохранена техническая терминология, но она спрятана в глубине IA.

## Что строится на следующих этапах

1. Углубление метрик (когорты, сегменты, unit-экономика, LTV-паттерны).
2. Расширение `Аналитики` и `Клиентов` до сценариев возврата и удержания.
3. Доработка `Рассылок` до продуктовой модели доходности кампаний.
4. Наращивание контекстных рекомендаций и action-driven UX.
5. Перевод оставшихся служебных экранов в бизнес-подачу, где это уместно.

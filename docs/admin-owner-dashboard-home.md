# Admin Owner Dashboard Home

## Цель шага

Усилить только экран `Главная` до owner-grade уровня: за 3 секунды показывать деньги, динамику, потери и следующее действие без перегруза интерфейса.

## Что добавлено на экран `Главная`

1. `Hero KPI` (money-first):
- выручка сегодня;
- оплаченные/всего заказы сегодня;
- средний чек сегодня;
- повторные заказы и повторные клиенты;
- конверсия заказ -> оплата.

2. `Сравнение периода`:
- к вчера (до текущего времени дня);
- к 7-дневному среднему;
- к аналогичному дню прошлой недели;
- с короткой интерпретацией «лучше/хуже».

3. `Требует внимания` (action-карточки с приоритетами):
- неоплаченные заказы;
- обращения без первого ответа;
- недоставки рассылок;
- динамика повторных продаж.

4. `Потери прибыли`:
- сумма замороженной выручки в неоплаченных заказах;
- недоохват рассылок;
- риск в поддержке (ожидание ответа).

5. `Точки роста`:
- повторные продажи;
- спящие клиенты (30+ дней без заказа);
- качество последней рассылки относительно среднего.

6. `Инсайт дня`:
- rule-based блок с человеческим пояснением, что именно важно сегодня.

7. `Быстрые действия`:
- крупные переходы в заказы, клиентов, рассылки, поддержку, аналитику.

8. `Мини-графики`:
- выручка по дням (7 дней);
- заказы по дням (7 дней).

## Используемые метрики и расчеты

## Деньги

- `revenueToday`: сумма оплаченных/завершенных заказов за сегодня.
- `revenue7d`: выручка за 7 дней.
- `revenue30d`: выручка за 30 дней.

Правило выручки заказа:
- берем `total_paid`, если > 0;
- иначе берем `total`, если статус относится к оплачиваемым (`PAID`, `COMPLETED`, `DELIVERED`);
- иначе в выручку не включаем.

## Заказы

- `totalToday`: всего заказов сегодня.
- `paidToday`: оплаченных заказов сегодня.
- `unpaidToday`: неоплаченные заказы сегодня.
- `paymentConversionTodayPct`: `paidToday / totalToday * 100`.

## Средний чек

- `avgCheckToday`: `revenueToday / paidToday`.
- дельты считаются к:
  - вчерашнему окну до текущего времени;
  - 7-дневному среднему;
  - аналогичному дню прошлой недели.

## Повторные продажи

- `repeatOrdersToday`: заказы сегодня от клиентов, у которых есть заказы до начала сегодняшнего дня.
- `repeatClientsToday`: уникальные клиенты с повторными заказами сегодня.

## Рассылки

- `latestCampaign`: последняя кампания с delivered/failed/blocked/total и `errorRatePct`.
- сравнение качества последней кампании со средним error-rate последних кампаний.

## Поддержка

- `activeThreads`: активные диалоги (`OPEN`, `PENDING`).
- `waitingWithoutResponse`: диалоги без первого ответа более часа.
- `avgFirstResponseMinutes7d`: среднее время первого ответа за 7 дней (в минутах).

## Потери

- `totalFrozenRevenue`: сумма неоплаченных заказов сегодня.
- недоставки рассылок за сегодня.
- обращения без первого ответа.

## Insight rules (rules-based engine)

Порядок приоритетов:

1. Если замороженная выручка в неоплаченных заказах высокая -> инсайт про возврат денег в кассу.
2. Если много обращений без ответа -> инсайт про сервисный риск.
3. Если выручка заметно выше вчера -> инсайт про позитивный рост.
4. Если чек падает, но заказов больше -> инсайт про компенсацию объемом.
5. Если последняя рассылка лучше среднего -> инсайт про рабочий маркетинговый паттерн.
6. Иначе -> нейтральный инсайт «день идет ровно».

Правила прозрачные, без ML и без «черного ящика».

## Attention rules

Каждая карточка имеет:
- заголовок проблемы;
- бизнес-описание;
- размер эффекта (`impactLabel`);
- приоритет: `critical`, `important`, `watch`;
- CTA с переходом.

## Action-переходы с главной

С главной внедрены переходы с предфильтрами:

- неоплаченные заказы -> `orders` + `orderFilter=unpaid`;
- клиенты без ответа -> `support` + `supportFilter=waiting`;
- ошибки рассылок -> `broadcasts` + `broadcastsFilter=RUNNING`;
- спящие клиенты -> `clients` + `clientFilter=sleeping`;
- быстрые действия -> соответствующие разделы (в том числе `analytics`).

Механизм реализован через компактный action payload: `screen + filters`.

## Backend contract

Расширен `GET /api/admin/mobile-summary` без ломки существующего маршрута:

- добавлены блоки:
  - `money`
  - `orders`
  - `support`
  - `broadcasts.latestCampaign`
  - `hero` (owner KPI)
  - `comparison` (3 среза)
  - `attention`
  - `losses` (с агрегатом и item-списком)
  - `growthPoints`
  - `quickActions`
  - `insight` (ruleId/tone/title/message)
  - `charts` (`revenue7d`, `orders7d`, `avgCheck7d`)

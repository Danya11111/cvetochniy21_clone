# Экран "Аналитика" как decision screen (mobile)

## Цель

Экран `Аналитика` реализован как слой управленческих решений, а не как набор графиков:
- показывает динамику бизнеса;
- объясняет, что растет и что проседает;
- выделяет точки роста и тревожные зоны;
- дает переходы к действиям в операционных экранах.

## Структура экрана

Реализация:
- frontend: `frontend/admin/app.js`
- backend: `backend/admin-repository.js`, `backend/admin-routes.js`

Секции экрана:
1. Переключатель периода: `Сегодня`, `7 дней`, `30 дней`, `90 дней`.
2. Executive summary периода (выручка + ключевые KPI).
3. Главные тренды (карточки сравнений).
4. Точки роста.
5. Тревожные зоны.
6. Rules-based business insights.
7. 5 бизнес-блоков:
   - деньги,
   - заказы,
   - клиенты,
   - рассылки,
   - поддержка.
8. Short-form charts:
   - выручка по дням,
   - заказы по дням,
   - средний чек по дням.
9. Блок "Что делать дальше" (action cards).

## Analytics summary endpoint

Добавлен endpoint:
- `GET /api/admin/analytics/summary?period=today|7d|30d|90d`

Возвращает DTO:
- `period`
- `totals`
- `sections` (money/orders/clients/broadcasts/support)
- `comparisons`
- `charts`
- `growthSignals`
- `riskSignals`
- `insights`
- `actionTargets`

## Какие comparisons считаются

Для выбранного периода считается сравнение с предыдущим аналогичным периодом:
- выручка
- заказы
- средний чек
- повторные продажи
- lost reach (рассылки)
- waiting support (поддержка)

Для каждой метрики:
- `current`
- `previous`
- `delta`
- `deltaPct`
- `interpretation`

## Как формируются insights

Rules-based, короткие, бизнес-ориентированные:
- выручка растет быстрее заказов -> усиление среднего чека;
- заказов больше, но средний чек ниже -> риск маржи;
- повторные продажи растут -> retention усилился;
- потерянный охват растет -> проблема в канале рассылок;
- waiting support растет -> сервисное давление.

Если выраженных сигналов нет, отдается нейтральный insight.

## Action-переходы

Из аналитики работают переходы в:
- `Заказы` (неоплаченные / проблемные),
- `Клиенты` (спящие / повторные / high-value),
- `Рассылки` (problematic / repeatable),
- `Поддержка` (waiting).

Переходы реализованы через существующий action payload (`apply-nav-action`).

## Принципы визуального слоя

- mobile-first секции с короткими блоками;
- сначала выводы и контекст, потом графики;
- только компактные графики без перегруза;
- понятные tone/badge для трендов и инсайтов.

## Ограничения этапа

На этом шаге не реализованы:
- полноценный BI-модуль;
- когортная и предиктивная аналитика;
- глубокая маркетинговая атрибуция выручки.

Сделан фокус на decision-oriented mobile analytics с честными прокси и рабочими action links.

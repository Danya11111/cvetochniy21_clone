# Admin Orders Mobile

## Назначение экрана

`Заказы` — операционный центр прибыли:
- показывает деньги и риски в заказах;
- выводит приоритетные заказы без открытия карточки;
- дает быстрые действия в 1-2 тапа.

Экран проектирован mobile-first, без desktop-таблиц как основного формата.

## Структура экрана

1. Верхний summary:
- сколько заказов в текущем срезе;
- сколько неоплаченных, срочных и проблемных.

2. KPI strip (кликабельные):
- `Всего`
- `Оплачено`
- `Не оплачено`
- `Срочные`
- `Проблемные`

3. Быстрые фильтры:
- `Все`
- `Сегодня`
- `Ждут оплаты`
- `Проблемные`
- `Срочные`
- `Оплачены`
- `Повторные`
- `Крупные`
- `На завтра`
- `Без внимания`

4. Контекстный блок:
- `Требует внимания по заказам`
- `Потенциально замороженная выручка`
- highlights проблемных заказов.

5. Список карточек заказов:
- бизнес-подача в карточках;
- быстрый скан по сумме, риску, срочности и клиенту.

## Карточка заказа: что видно сразу

- номер заказа и клиент;
- сумма;
- статус;
- attention-уровень (`Срочно`, `Важно`, `В норме`);
- срочность;
- признак `Новый/Повторный`;
- признак `Высокий чек`;
- доставка (дата/время);
- время создания;
- число позиций;
- число заказов клиента и LTV stub;
- короткий бизнес-подзаголовок (`order_subtitle`/reason).

## Действия из карточки

- `Открыть` (раскрытие текущих деталей заказа);
- `Клиент` (переход в раздел клиентов с предзаполненным поиском по telegram id);
- `Тема` (быстрый переход в Telegram topic при наличии).

## Поддерживаемые фильтры и deep-link state

Поддерживается `orderFilter`:
- `all`
- `today`
- `paid`
- `unpaid`
- `urgent`
- `problematic`
- `repeat`
- `large`
- `tomorrow`
- `no_attention`

Главная может передавать фильтр через action payload (`screen + filters`), и экран заказов применяет его при открытии.

## Derived fields (backend)

На `GET /api/admin/orders` добавлены derived-поля (совместимо со старым контрактом):

- `amount_kopecks` (сумма заказа для UI, целые копейки)
- `is_paid`
- `is_unpaid`
- `is_large_order`
- `is_repeat_client`
- `is_new_client`
- `client_orders_count`
- `client_lifetime_value_stub`
- `items_count`
- `delivery_bucket` (`today` / `tomorrow` / `later` / `unspecified`)
- `is_urgent`
- `attention_level` (`critical` / `important` / `normal`)
- `attention_label`
- `attention_reason`
- `is_problematic`
- `order_subtitle`
- `topic_link`

## Правила проблемного/срочного заказа

Rules-based logic:

- `critical`:
  - не оплачен + срочная доставка;
  - не оплачен + высокий чек.
- `important`:
  - не оплачен;
  - долго без движения;
  - проблемный статус;
  - повторный клиент без подтверждения оплаты.
- `normal`:
  - остальные заказы.

Срочность:
- доставка сегодня;
- либо доставка завтра утром (эвристика по времени).

## Backend изменения

Изменены/добавлены:

1. `GET /api/admin/orders`
- расширен DTO полями для mobile order cards.

2. `GET /api/admin/orders/:id`
- аналогично расширен order-блок в detail response.

3. `GET /api/admin/orders/summary` (новый)
- агрегаты для order screen:
  - totals по ключевым сегментам;
  - frozen revenue;
  - order signals;
  - highlights проблемных заказов.

## Связь с Главной

Главная и Заказы связаны через единый action/filter contract:
- из Home по кнопкам/attention можно открыть `orders` с нужным фильтром;
- экран заказов сохраняет mobile-first UX и дополняет Home операционной глубиной.

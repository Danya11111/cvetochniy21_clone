# Production smoke-test (после деплоя)

Практический чеклист для оператора. Все шаги на **боевом или staging-домене** с заполненным env (`deploy/env.example` как образец). Не публикуйте токены и пароли в чаты — секреты только на сервере.

---

## Перед началом

- Приложение запущено (`systemd` / ваш процесс-менеджер).
- Заполнены **`APP_PUBLIC_URL`**, **`TELEGRAM_BOT_TOKEN`**, **`TELEGRAM_WEBHOOK_SECRET`**, интеграции (**Т‑Банк**, **МойСклад** по необходимости).
- **`ADMIN_OWNER_TG_ID`** указывает на владельца (единственный, кто управляет списком админов через Mini App).
- Для брошенных корзин: при включении **`ABANDONED_CARTS_ENABLED=1`** задайте **`TELEGRAM_TOPIC_ABANDONED_CARTS_ID`** (нет темы → Telegram-уведомления не уходят, данные в БД сохраняются).

---

## Чеклист

1. **Открыть сайт** — главная мини-приложения / storefront загружается без ошибки сертификата и без SPA-падений (`/__SPA__/нет смешения storefront/admin`).
2. **Каталог** — `/api/products` отдаёт JSON; карточки товаров на главной отображаются.
3. **Корзина** — добавить товар; суммы пересчитываются.
4. **`cart_key`** — после добавления в DevTools → Application → Local Storage есть ключ наподобие `f21_cart_key_v1` с UUID; повторный визит не генерирует новый ключ без причины.
5. **Abandoned sync не ломает UI** — в консоли сети есть необременящие POST `/api/abandoned-cart/sync` (ответ можно игнорировать); добавление товаров работает даже при 503 этого endpoint (**fail-open** задуман во storefront-клиенте).
6. **Checkout старт** — открыть модальное окно оформления; поля доступны; **не должно быть** блокирующего падения при ошибке синка брошенной корзины (клиент ожидается **fail-open**).
7. **Тестовый заказ** — оформить без оплаты или до экрана оплаты по вашему сценарию; локально в БД строка в `orders`, статус **`PENDING_PAYMENT`** / след. по логике вашего сценария (**не ожидайте** полного PAID без оплаты).
8. **Заказ в админке** — Mini App admin → заказы / старые экраны показывают созданный заказ (**права** по initData).
9. **МойСклад при ошибке не ломает checkout** — симулируйте недоступность MS (env off / неверный token на тесте): ответ checkout должен включать предупреждение (`moysklad_degraded`), а **не 500 с пустым телом без пояснения**; пользователь может перейти к оплате если сервер отдал `paymentUrl`.
10. **Payment URL Т‑Банка** — в ответе есть ссылка оплаты для нового заказа; домен в ссылке консистентен с конфигом (**не** случайный localhost в production).
11. **Success/Fail callback** — после тестовой оплаты статус заказа переходит в **`PAID`** или при отказе — **`PAYMENT_FAILED`** (не смешивать с «отменой заказа» в UI).
12. **Telegram заказ** — в теме заказов темы приходит уведомление (**если** включено `ORDERS_TOPIC_NOTIFICATIONS_ENABLED` и топики настроены).
13. **Брошенные корзины в Telegram** — при включённом сервисе и настроенном **`TELEGRAM_TOPIC_ABANDONED_CARTS_ID`** после сценария «положили в корзину → ушли или долго на checkout» приходит уведомление (**не** для `recovered`/`cleared`/`expired`).
14. **Админка** — вход по WebApp initData работает; дашборд загружается (**HTTP 200** `/api/admin/dashboard-v2`).
15. **Owner и админы** — под владельцем (**`ADMIN_OWNER_TG_ID`**) открывается блок «Администраторы», можно добавить тестового ID и удалить (**кроме владельца**).
16. **Обычный админ** — под аккаунтом без owner: **нет** успешного POST/DELETE `/api/admin/admins` (**403 OWNER_ONLY**).
17. **Нет UI отмены/возврата заказа** — на витрине нет кнопок «отменить заказ» / «оформить возврат»; в админке нет метрик «возвраты после оплаты» из прошлой версии.
18. **Карусель топ-товаров** — в **mini-admin** дашборде блок «Топ популярных товаров»: изображения видны, стрелки листают горизонтальный скролл через `scrollBy`, у стрелок **aria-label** «Предыдущие товары» / «Следующие товары»; при отсутствии горизонтального переполнения стрелки скрыты/неактивны. На **публичной витрине** отдельной карусели «топ» в текущей вёрстке нет (только общий каталог).
19. **Миграция validate** — на новом сервере: `SQLITE_DB_PATH=/path/to/db.sqlite bash scripts/migration/validate.sh` → `integrity_check` **ok**, в списке есть **`abandoned_carts`**.
20. **Домен и логи** — в конфиге/ответах нет старого хардкода прод-домена (кроме намеренных примеров в `.example`); в логах нет циклических 500 на `/api/checkout` и дублей фатальных ошибок webhook Т‑Банка при повторной доставке (**идемпотентность** на стороне обработчика).

## Роллбек
